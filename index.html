o-syntax v1

naming conventions for the o-internet.

#Normal Syntax

o-syntax was designed for basic file organization for human and computer reading. The use of this syntax is to allow the computer and user to read metadata and use it for quick file finding.

fig 1.1

[author]-content_name-[version] | [volume].[extension]

#Hypehenation

This is what is used to designate each part of the name.

fig 2.1

[author]-content_name-[volume].[extension]

#Defining Authorship

An author of any form of content shall delineate their authorship by the use of prefixing the title of the project with a symbolic representation of their name.

fig 3.1

owebboy-content_name

o-content_name

oliver_pope-content_name

#Versioning (Volumes)

The version of a file determines the file's reputation and age in the project or database (4.1). An alternate form of organization would be volumes to show file's recency (4.2).

To help reduce redundancy, the file's version may use a file hierarchy characterized by a period.

fig 4.1

[author]-content_name-1.0

fig 4.2

[author]-content_name-IX

[author]-content_name-2

#Extensions

Extensions come at the end of the file and show the file's content-type
